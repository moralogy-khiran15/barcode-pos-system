システム開発作業報告書
日付: 2026年 3月 19日
作成: ひらのさま ＆ Antigravity

本日は、2月27日に立ち上げた「バーコード読込システム（POSレジアプリ）」の本格的な機能拡張を行いました。ローカルストレージのみだったプロトタイプを、Googleスプレッドシート連携による実用的なシステムへと進化させ、会計区分別の売上管理機能まで実装しました。

1. 本日の主要成果（Achievements）

🔧 ビルドエラー・バグ修正

Tailwind CSS v4へのPostCSS設定の互換性修正、およびReact Strict Modeで数量が2重カウントされるバグを解消しました。

📊 Googleスプレッドシート連携

Google Apps Script (GAS) を中継役として、商品マスター・売上履歴をスプレッドシートで共有管理できる仕組みを構築しました。

🏷️ 会計区分機能

商品登録時に会計区分をプルダウンから選択できるようになり、お会計確定時には区分ごとに自動で別シートに振り分けて記録されます。

⚡ スキャン安定性向上

連続スキャン防止の時間を延長し、GAS通信中のスキャンブロック機能を追加。バーコードリーダーの誤認識を大幅に削減しました。

2. 実施内容の詳細（Chronological Log）

前半：環境修正とバグ対応

  • ビルドエラー修正： Tailwind CSS v4で @tailwindcss/postcss が必要になったため、postcss.config.jsとindex.cssを修正しました。
  • 数量2重カウント修正： setInventoryの中にsetScannedItemsが入れ子になっていた構造を、useRefを使ったリファクタリングで解消しました。

中盤：Googleスプレッドシート連携の構築

  • GASバックエンド構築： doGet（商品マスター取得）とdoPost（売上記録・商品登録）のAPIエンドポイントを作成しました。
  • アプリ側の通信実装： fetchを使い、アプリ起動時のマスター取得、お会計確定時の売上送信、新規商品のマスター登録を実装しました。
  • デプロイ管理の学習： GASのバージョン管理（新バージョンの作成とデプロイ）の手順を確立し、コード更新の反映フローを確認しました。

後半：会計区分機能の実装

  • 区分マスターの外部管理： 「Categories」シートを新設し、区分名をスプレッドシート上で自由に追加・変更できるようにしました。
  • プルダウン選択UI： 商品登録画面の会計区分を手入力からプルダウン選択式に変更しました。
  • 売上シート自動振り分け： お会計確定時に「Sales_飲料」「Sales_医薬品」等、区分名ごとにシートが自動作成・振り分けされる機能を実装しました。
